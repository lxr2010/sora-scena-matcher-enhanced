# 空之轨迹1st 与 空之轨迹FC进化版 的台词校对脚本
最近为了更好地应用大模型的能力，对原来的[sora-scena-matcher](https://github.com/lxr2010/sora-scena-matcher)进行了重构。
重构后的脚本应该可以用于将来的空之轨迹2nd，甚至零之轨迹和碧之轨迹。不过届时作者可能没有精力维护，需要使用的请随意fork。

## 特点
* 位置敏感哈希算法 + 基于锚点的优化 + 最小编辑距离匹配
* 保留多候选项匹配
* 匹配项rapidfuzz WRatio分数超过92
* 复杂匹配场景通过大模型预测候选项
* 处理相同的重置版与EVO台词，成功匹配数达到28425，接近人工校对水平（27537）
* 精度相比原始版本有明显提高
* 识别片假名、轨迹系列专有名词、ED6旧引擎Gaiji
* 无pytorch/GPU需求

## 使用方法
* 输入：
  * `scena_data_jp_Command.json`, 空之轨迹1st解包得到的地图和脚本文件。
  * `script_data.json`, 空之轨迹FC进化版的脚本文件。
  * `scena_data_sc_Command.json`, 空之轨迹1st简体中文版解包得到的地图和脚本文件。
* 输出：
  * `match_result.csv`, 匹配结果。
  * `llm_*.json`, 大模型调用的结果缓存，避免相同输入多次触发大模型调用，造成Token浪费。
  * `*.json`, 匹配过程中的中间结果。`main.py`中定义了几个功能函数，它们每次执行都会通过json格式将中间结果序列化到文件中。首次执行结束后，可以注释掉这些功能函数，直接加载中间结果文件，节约时间。

* 注意事项：
  * 大模型调用默认使用gpt-4o-mini模型，完整无缓存运行，对于空之轨迹1st，需要约1100次调用。没有针对其他模型进行优化，好在gpt-4o-mini价格便宜。换用其他模型请注意开销。
  * 大模型环境可以通过复制`.env.example`文件为`.env`，填入自己的端点信息进行配置，只需要兼容OpenAI API格式即可。选择的模型最好有较好的格式化输出能力，因为几乎所有使用的大模型调用都使用JSON格式返回，如果没有良好的支持容易导致解析错误。
  * 大模型预测内容和多候选项的备选内容都以附注（`Annotation`字段）的形式添加到`match_result.csv`。
